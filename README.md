# ai-learning-log

AI 활용 / Claude Code 학습 로그.

AI와 대화하며 배운 것을 **그날그날 실시간으로 기록**하는 살아있는 노트입니다.

## 원칙 (Andrej Karpathy, ["전문가가 되는 법"](https://x.com/karpathy/status/1325154823856033793))

1. 구체적인 프로젝트를 반복해서 맡아 **깊이, 끝까지** 해낸다 — 필요한 것을 **그때그때(on demand)** 배우고, 기초부터 넓게 훑지 않는다
2. 배운 것을 전부 **내 언어로, 남을 가르치듯** 정리한다
3. 남이 아니라 **과거의 나**와만 비교한다

이 노트는 그래서 세 단계로 돈다:
**실전에서 걸린 문제를 파고들어 확인** (1) → **노트로 적립하고, 쌓이면 블로그에 가르치듯 발행** (2) → **정리 세션마다 "예전의 나는 뭘 몰랐나" 회고** (3)

## 이 노트의 규칙

- 이해 못 한 채로 넘어가지 않는다 — 질문이 꼬리를 물면 그대로 기록한다
- 답을 듣기 전에 **먼저 예상**하고, 답은 **내 말로 다시 표현**한 것만 적는다 (AI 문장 복붙 금지)
- 추상 설명보다 **실제로 해보고 확인한 것**을 적는다 — 요약 콘텐츠가 아니라 [1차 소스](https://x.com/karpathy/status/1756380066580455557)를 본다
- 같은 질문이 다시 나오면 기존 노트부터 찾는다 — 질문과 답이 쌓여 다음 세션이 혜택을 보는 구조(컴파운딩)
- 실패·삽질도 기록한다 (원인을 파고든 과정이 제일 남는다)
- 자주 헷갈리는 지점은 ⚠️ 로 표시한다
- 새 내용은 항상 파일 끝에 **추가(append)** 한다 — 과거 기록은 고치지 않는다. 틀린 이해는 `(정정: 노트 N)` 제목의 새 노트로 바로잡는다
- 노트가 15개 쌓일 때마다 "깨달은 것"을 갱신하고 "예전의 나는 몰랐던 것" 한 줄을 회고한다
- 노트는 [블로그](https://mutpp.tistory.com)에도 `[AI 학습로그]` 시리즈로 **같은 내용**을 발행하고, 발행본 마크다운은 `blog/` 폴더에 보관한다

## 노트 템플릿

~~~markdown
### 노트 N — 깨달음 한 줄 제목 (YYYY-MM-DD)

**상황**: 무엇을 하다가 왜 걸렸나 (1~2줄)

**예상**: 실험 전 내 예상 (선택 — 깨진 예상이 제일 좋은 노트감)

**확인한 것**:
```bash
실제 실행한 명령과 결과
```
설명 문단 (필요한 만큼)

**결론**: 한 줄 요약

**써먹기**: 실무 어디에 적용할 수 있나 (선택)

⚠️ 헷갈렸던 지점 (있을 때만)
~~~

## /study-log 스킬

이 저장소에는 노트 적립을 자동화하는 Claude Code 스킬이 들어 있다: [.claude/skills/study-log/SKILL.md](.claude/skills/study-log/SKILL.md)

저장소를 클론해서 Claude Code로 열고 "오늘 배운 거 기록해줘"라고 하면 —
복습 퀴즈 1개(간격 반복) → 노트 추출·작성 → 질문 목록/로드맵 갱신 → 15개마다 회고 제안 → 블로그 발행(선택) → 커밋·푸시까지 이어서 해준다.
블로그 관련 설정만 본인 것으로 바꾸면 된다.
