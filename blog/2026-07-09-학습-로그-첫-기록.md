오늘부터 'AI 활용 학습 로그'를 시작했다. AI와 대화하며 배운 것을 그날그날 GitHub에 노트로 쌓는 방식이다.

## 방식

Andrej Karpathy의 "전문가가 되는 법"(2020년 트윗)을 따른다. 원문의 뉘앙스를 살리면 이렇다.

1. 구체적인 프로젝트를 반복해서 맡아 깊이, 끝까지 해낸다 — 필요한 것을 그때그때(on demand) 배우고, 기초부터 넓게 훑지 않는다
2. 배운 것을 전부 내 언어로, 남을 가르치듯 정리한다
3. 과거의 나와만 비교하고, 남과는 비교하지 않는다

그래서 이 학습은 세 단계로 돈다. 실전에서 걸린 문제를 파고들어 직접 확인하고, GitHub 노트에 내 언어로 적립한 뒤, 이 블로그에 가르치듯 풀어 쓴다. 노트가 쌓이면 예전의 나는 뭘 몰랐는지 돌아본다. 이 글이 그 "가르치듯 풀어 쓰기"의 첫 번째 글이다. 실패와 삽질도 기록한다.

## 첫날 배운 것 세 가지

### 1. git 인증 정보는 세 군데에 흩어질 수 있다

git config의 user.password, ~/.git-credentials, gh CLI 자체 로그인 — 전부 다른 곳이다. git push가 되는 것과 gh가 되는 것은 별개의 인증이다. 내 경우 git config에 만료된 토큰이 평문으로 박혀 있었다. 바로 삭제했다.

### 2. gh 로그인은 토큰 스코프가 발목을 잡는다

gh auth login은 토큰에 read:org 스코프가 없으면 거부한다. 하지만 REST API 직접 호출은 그 스코프 없이도 된다. "gh가 안 됨"과 "토큰이 죽음"은 다르니, API로 토큰부터 검증할 것.

### 3. Claude Code 스킬의 실체는 마크다운 파일 하나다

학습 노트를 자동으로 적립하는 /study-log 스킬을 직접 만들어봤다. SKILL.md 파일 하나가 전부이고, frontmatter의 description이 곧 트리거다. 모델이 사용자의 말과 description을 대조해서 스스로 호출을 결정하기 때문에, 기능 설명보다 "사용자가 어떤 말을 하면 써라"를 적어야 잘 발동된다.

재미있었던 건, "새 스킬은 세션을 재시작해야 뜬다"고 예상했는데 틀렸다는 것. 저장하자마자 같은 세션에서 바로 인식됐다. 예상과 실제가 다를 수 있으니 항상 직접 확인할 것 — 이게 이 학습 로그의 핵심 루프다.
