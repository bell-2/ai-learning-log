# tistory-post — 티스토리 발행

- 블로그: `mutpp.tistory.com` (bellog) — **클론해서 쓰는 경우 본인 블로그로 교체**. Tistory Open API는 종료됨(2024-02) — **Claude in Chrome 브라우저 조작으로만 발행 가능**. 브라우저가 연결되어 있지 않으면 사용자에게 연결을 요청한다.
- 입력: 제목, 본문(마크다운), 카테고리(기본: AI 하위 `Claude`), 공개 범위(기본: 공개). 글의 내용 규칙(제목 형식, 구성)은 호출하는 쪽 책임 — 이 스킬은 발행 공정만 담당한다.

## 절차

1. 글쓰기 페이지: `<블로그>/manage/newpost`. 기존 글 수정은 글 관리(`/manage/posts/`)의 수정 링크(`manage/post/<번호>`)로 진입.
2. 카테고리 선택 → 제목 입력.
3. 편집기를 **HTML 모드**로 전환 (우상단 모드 드롭다운). ⚠️ 전환 시 네이티브 confirm 창이 떠서 브라우저 자동화가 멈춘다(클릭이 타임아웃돼도 클릭 자체는 들어간 것) — **사용자에게 직접 확인 버튼을 눌러달라고 요청**할 것.
4. 본문(마크다운)을 **개행 문자가 전혀 없는 한 줄짜리 HTML**로 변환해 실제 타이핑(computer type)으로 입력. `<pre><code>` 안의 줄바꿈은 `<br>`로, 코드 들여쓰기는 공백 그대로 넣는다.
   - ⚠️ 편집기에 JS로 setValue 주입 금지 — 티스토리는 실제 입력 이벤트로만 내부 모델을 동기화하기 때문에, 주입하면 **원문 텍스트가 변환 없이 그대로 발행**된다 (개행 다 뭉개진 한 덩어리로 보임).
   - ⚠️ 마크다운 모드에 여러 줄을 타이핑하는 것도 금지 — CodeMirror 자동 들여쓰기가 줄마다 앞 줄 들여쓰기를 물려받아 **코드블록이 계단식으로 망가진다**. 개행 없는 한 줄 HTML은 이 문제가 원천적으로 없다.
   - 입력 후 편집기 값(글자 수·태그 수)을 JS로 **읽어서**(읽기는 안전) 검증한다.
   - Mermaid 등 티스토리가 렌더하지 못하는 요소는 이미지 캡처로 교체하거나 빼고 발행한다.
5. "완료" 버튼 → 발행 창에서 **공개** 선택 (⚠️ 기본값이 비공개) → **사용자에게 최종 확인 후** "공개 발행" 클릭.
6. 발행 후 글 관리(`/manage/posts/`)에서 카테고리 확인, 아니면 체크박스 → 변경 → 카테고리 변경으로 이동.
7. **발행된 글 페이지를 직접 열어 렌더링 검증**: 본문 DOM에 `h2`/`pre`/`table`이 실제로 있고 `**`, ``` 같은 마크다운 원문이 새지 않았는지 확인한다. 원문이 그대로 보이면 변환이 안 된 것 — 글 수정으로 들어가 4번 방식으로 다시 입력한다.

## 금지 사항

- 공개 발행 버튼은 사용자 확인 없이 절대 누르지 않는다.
- 발행 전 본문에 비밀번호·토큰 등 민감정보가 없는지 확인한다.
